### CVE-2026-31431 Patch (Lee auf kyberguard-vm ausfuehren)

```bash
# Schritt 1: Aktuellen Kernel pruefen
uname -r

# Schritt 2: Updates einspielen (4 restliche Pakete vom Morgen)
sudo apt update && sudo apt upgrade -y

# Schritt 3: Kernel-Version nach Update pruefen
apt-cache show linux-image-generic | grep Version

# Schritt 4: Wenn neuer Kernel: Reboot planen
# (Empfehlung: Reboot in Wartungsfenster -- Sonntag 03:00 Uhr)
sudo shutdown -r +1 "Kernel-Update Wartung"

# Schritt 5: Nach Reboot -- CVE-Pruefung
uname -r
# Erwartet: 6.8.0-XXX-generic (neuer als 6.8.0-90)
```

Hinweis: Reboot beider VMs (kyberguard-vm + frieguen-hub) ist seit Morgen empfohlen (offene Updates). Kombinieren.

### AmneizaWG-DKMS Fix (Kernel-Update-Resilienz)

```bash
# Auf frieguen-hub (nach Reboot-Wartungsfenster):
sudo apt install -y dkms build-essential linux-headers-$(uname -r)

# AmneizaWG DKMS-Modul:
git clone https://github.com/amnezia-vpn/amneziawg-linux-kernel-module.git
cd amneziawg-linux-kernel-module
# DKMS-Build (sudo noetig):
sudo dkms add .
```

### Runbooks: 5 kritische Infra-Szenarien

**Szenario 1 -- kyberguard-vm nicht erreichbar**
```
1. Hetzner Cloud Console oeffnen (console.hetzner.cloud)
2. VM-Status pruefen: Running/Off?
3. Wenn Off: VM starten
4. Wenn Running aber kein SSH: Console-Login nutzen
5. Logs: journalctl -xe | tail -50
6. Docker-Stack pruefen: docker ps -a
7. Wenn alle Container laufen: nginx reload (sudo systemctl reload nginx)
8. Wenn Container down: docker compose up -d (im jeweiligen Verzeichnis)
```

**Szenario 2 -- FastAPI-Backend antwortet nicht (502 Bad Gateway)**
```
1. SSH auf kyberguard-vm
2. docker ps -a | grep kyberguard
3. Wenn Container Exit: docker logs kyberguard-api --tail 50
4. Neustart: cd /home/ceuleeneo && docker compose up -d
5. Health-Check: curl -s http://172.18.0.1:8000/health
6. Wenn Datenbank-Fehler: docker logs postgres-kyberguard --tail 20
```

**Szenario 3 -- VPN-Verbindung zu frieguen-hub tot (awg0 down)**
```
1. SSH auf frieguen-hub via direkter IP (162.55.217.13)
2. Pruefe: sudo systemctl status awg-quick@awg0
3. Wenn inaktiv: sudo systemctl start awg-quick@awg0
4. Wenn Kernel-Modul fehlt (nach Kernel-Update):
   -- Rescue-Modus (Hetzner Console) oder DKMS-Fix (s.o.)

```

**Szenario 4 -- TLS-Zertifikat laeuft ab (Frueherkennung)**
